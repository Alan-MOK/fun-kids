# Fun Kids - 幼儿互动教育平台

## 项目概述

为 4-6 岁儿童打造的 PC 浏览器端互动教育 Web 应用。通过色彩丰富、趣味互动的方式帮助孩子认识世界、学习基础知识。部署到云服务器，通过域名或 IP 访问。

## 技术方案

- **后端**：Node.js + Express 5（Web 服务器）
- **数据库**：SQLite（单文件数据库，WAL 模式，零配置，便于备份迁移）
- **图片存储**：服务器本地 `uploads/` 目录，通过 API 上传和访问
- **语音方案**：百度 TTS API（文字转语音） + 本地 MP3 录音文件（声母/韵母发音）
- **访问控制**：密码保护，前端和管理后台分别设置独立密码，JWT 鉴权
- **前端**：Tailwind CSS + Vanilla JS + Font Awesome（本地部署） + 百度 TTS API
- **中文字体**：Noto Sans SC（本地部署）
- **图标库**：Font Awesome（本地部署在 `public/libs/fontawesome/`）

## 访问控制

### 两套密码，两个角色

| 角色 | 密码用途 | 可访问范围 |
|------|---------|-----------|
| **前端用户**（孩子/家长） | 进入学习页面 | 主页、拼音学习页面（只读） |
| **管理员**（家长） | 进入管理后台 | 以上全部 + 管理后台（读写） |

### 登录流程

1. 用户打开任意页面 → 如果未登录，显示**密码输入界面**（全屏居中）
2. 输入密码 → 调用 `POST /api/auth/login` 验证
3. 验证成功 → 服务端返回 JWT token（含角色信息），前端存入 localStorage
4. 后续所有 API 请求自动带上 `Authorization: Bearer <token>`
5. Token 有效期 7 天，过期后需重新输入密码

### 密码配置

密码在服务端配置文件中设置：

```javascript
// config.js
module.exports = {
  FRONTEND_PASSWORD: 'kids123',              // 前端访问密码（给孩子用，简单好记）
  ADMIN_PASSWORD: 'admin456',                // 管理后台密码（给家长用）
  JWT_SECRET: 'fun-kids-secret-key-2026',    // JWT 签名密钥
  PORT: 3003,                                // 服务器端口
  BAIDU_TTS: {                               // 百度 TTS 配置
    API_KEY: '...',
    SECRET_KEY: '...',
    TOKEN: '...',                             // 需定期刷新（30天有效期）
    PER: 3,                                   // 默认音色：百度逍遥（男声）
  },
};
```

- **管理后台登录**：简洁输入框 + "管理员登录"标题，专业风格

## API 设计

### 认证接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/auth/login` | 登录验证，body: `{ password, role: "user"/"admin" }`，返回 JWT token |

### 配置接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/config/tts` | 获取百度 TTS 配置（Token、音色） | 前端 |

### 拼音数据接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/pinyin` | 获取全部拼音数据（支持 `?category=动物&hasImage=true&search=猫&initial=m&final=ao&tone=1`） | 前端 |
| GET | `/api/pinyin/:id` | 获取单条拼音数据 | 前端 |
| GET | `/api/pinyin/categories` | 获取所有分类列表 | 前端 |
| GET | `/api/pinyin/stats` | 获取统计信息（总数/有图片数/缺图片数） | 前端 |
| POST | `/api/pinyin` | 新增一条拼音数据 | 管理员 |
| PUT | `/api/pinyin/:id` | 修改一条拼音数据 | 管理员 |
| DELETE | `/api/pinyin/:id` | 删除一条拼音数据 | 管理员 |
| POST | `/api/pinyin/:id/image` | 上传图片（multipart/form-data） | 管理员 |
| DELETE | `/api/pinyin/:id/image` | 删除图片 | 管理员 |
| GET | `/api/pinyin/export` | 导出全部数据为 JSON | 管理员 |
| POST | `/api/pinyin/import` | 导入 JSON 数据（支持覆盖/合并模式） | 管理员 |
| POST | `/api/pinyin/reset` | 重置为默认数据 | 管理员 |

### 语音录音接口

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | `/api/voice` | 获取已上传的语音文件列表 | 前端 |
| POST | `/api/voice/:filename` | 上传语音录音文件 | 管理员 |
| DELETE | `/api/voice/:filename` | 删除语音录音文件 | 管理员 |

> 所有接口都需要有效 token，写操作额外需要管理员权限。

### 数据库表结构

```sql
CREATE TABLE pinyin (
  id TEXT PRIMARY KEY,          -- 唯一ID，如 "m-ao-1-猫"
  pinyin TEXT NOT NULL,         -- 带声调拼音，如 "māo"
  initial TEXT NOT NULL,        -- 声母，如 "m"（零声母为空字符串）
  final TEXT NOT NULL,          -- 韵母，如 "ao"
  tone INTEGER NOT NULL,        -- 声调 1-4
  char TEXT NOT NULL,           -- 汉字，如 "猫"
  meaning TEXT,                 -- 释义，如 "小猫咪"
  category TEXT,                -- 分类，如 "动物"
  emoji TEXT,                   -- Emoji，如 "🐱"
  image TEXT,                   -- 图片文件名（存在 uploads/pinyin/ 下）
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 3.2 交互逻辑

1. **选择声母**：点击声母按钮，按钮高亮选中，当前选择区显示
2. **选择韵母**：点击韵母按钮，按钮高亮选中，当前选择区显示
3. **选择声调**：点击声调按钮，按钮高亮选中
4. **点击"组合"按钮**：
   - 前端调用 `GET /api/pinyin?initial=m&final=ao&tone=1` 查询
   - 如果有匹配 → 展示区显示拼音分解、汉字、图片，播放撒花动画
   - 如果无匹配 → 温和提示"这个组合不存在哦，换一个试试吧"
5. **语音朗读**：
   - 百度 TTS API 朗读汉字和释义
   - 本地 MP3 录音播放声母和韵母发音
   - 朗读模式开关：开启后选择声母/韵母时自动播放发音
6. **试试手气**：随机推荐一个拼音组合
7. **清除按钮**：重置所有选择

#### 3.3 语音方案（双轨制）

| 语音来源 | 用途 | 格式 |
|---------|------|------|
| 本地 MP3 录音 | 声母发音、韵母发音 | `uploads/voice/initial-*.mp3` `uploads/voice/final-*-*.mp3` |
| 百度 TTS API | 汉字朗读、释义朗读 | 实时合成 |

语音播放顺序：汉字 → 声母录音 → 韵母录音 → 汉字 TTS → 释义 TTS

#### 3.4 图片展示方案

前端展示优先级：
1. **优先展示服务器图片**：`/uploads/pinyin/xxx.jpg`，以卡片形式展示（圆角、阴影、最大 280px）
2. **备选 Emoji 展示**：没有图片时，使用 Emoji 以 120px+ 大字号展示
3. **兜底方案**：无 Emoji 的汉字，使用汉字本身的超大艺术化展示（描边、阴影、彩色渐变）

#### 3.5 设计风格要求

- **配色**：活泼明快，大量使用暖色调（橙、黄、绿、天蓝）
- **圆角**：所有按钮和卡片使用大圆角 (16px-24px)
- **字号**：
  - 声母/韵母按钮：24px+，确保清晰易点击
  - 结果汉字：96px-128px 超大渐变文字显示
  - 配图：最大 280px
- **按钮尺寸**：最小 60x60px，方便儿童点击
- **动画效果**：
  - 按钮点击时弹跳效果 (scale)
  - 结果展示时淡入 + 缩放动画
  - 正确组合时撒花粒子效果（CSS 实现）
  - 毛玻璃背景模糊效果 (backdrop-filter)
- **背景**：浅色渐变 + 卡片式布局

#### 3.6 特殊交互细节

- **智能提示**：选择声母后，自动灰化不能与之组合的韵母（如 j/q/x 只能搭配 i/ü 系韵母）
- **随机推荐**："试试手气"按钮，随机推荐一个拼音组合
- **零声母支持**：可选择"∅"零声母，直接选韵母组合（如 ā → 啊）
- **朗读模式**：开关控制，开启后点击声母/韵母自动播放发音

#### 4.2 核心功能

**数据管理 (CRUD)**：
- **查看**：表格展示所有拼音条目，支持排序（升序/降序）
- **新增**：弹出模态框，填写声母、韵母、声调、汉字、释义、分类、Emoji，上传图片
- **编辑**：点击编辑按钮，弹出模态框修改已有条目
- **删除**：带确认提示的删除功能

**图片管理**：
- **上传本地图片**：通过文件选择器上传，调用 `POST /api/pinyin/:id/image`，服务端自动压缩保存（800×800, JPEG 85%）
- **图片预览**：表格中 40×40 缩略图展示
- **批量图片状态**：一眼看出哪些条目缺少图片

**数据导入导出**：
- **导出 JSON**：调用 `GET /api/pinyin/export` 下载全部数据
- **导入 JSON**：选择 JSON 文件，调用 `POST /api/pinyin/import`，支持"覆盖"和"合并"两种模式
- **重置默认数据**：调用 `POST /api/pinyin/reset`，恢复为种子数据（带确认提示）

**筛选与搜索**：
- 按分类筛选（动物、自然、家庭等）
- 按图片状态筛选（全部/有图片/缺图片）
- 关键词搜索（匹配拼音、汉字、释义）

#### 4.3 管理后台设计风格

- **简洁专业**：白色背景、灰色边框、清晰的表格布局
- **非儿童向**：正常字号、紧凑布局，提高信息密度
- **高效操作**：表格行内快速编辑
- **响应式**：适配笔记本和桌面显示器

### 七、后端服务 (server.js)

#### 7.1 依赖

```json
{
  "dependencies": {
    "express": "^5.2.1",
    "better-sqlite3": "^12.6.2",
    "multer": "^2.0.2",
    "sharp": "^0.34.5",
    "jsonwebtoken": "^9.0.3"
  }
}
```

- **express**：Web 服务器框架（v5）
- **better-sqlite3**：同步 SQLite 驱动，WAL 模式
- **multer**：处理文件上传（v2）
- **sharp**：图片压缩/缩放（上传时自动压缩到 800×800，JPEG 85%）
- **jsonwebtoken**：JWT 生成与验证

#### 7.2 启动逻辑

1. 加载 `config.js` 配置（密码、JWT 密钥、百度 TTS、端口）
2. 初始化 SQLite 数据库（WAL 模式），建表（如果不存在）
3. 检查数据库是否为空，如果为空则导入 `data/pinyin-seed.js` 种子数据
4. 确保 `uploads/pinyin/` 和 `uploads/voice/` 目录存在
5. 注册认证中间件（JWT 验证 + 权限检查）
6. 注册 API 路由（登录接口免验证，其余需 token）
7. 挂载 `public/` 为静态文件目录
8. 挂载 `uploads/` 为静态文件目录
9. 监听端口（默认 3003）

#### 7.3 图片处理

- 上传时通过 sharp 自动压缩：最大 800×800px，JPEG 质量 85%
- 文件命名：`{id}_{timestamp}.jpg`
- 旧图片自动删除（替换上传时）

#### 7.4 百度 TTS 配置

- Token 需每 30 天刷新一次
- 刷新脚本：`public/tools/oauth2.0token.py`
- 默认音色：百度逍遥（PER=3，男声情感音色）

## 设计原则

1. **安全第一**：无外部链接、无广告、密码保护访问
2. **容错友好**：无效操作给予温柔提示，永远不显示错误信息
3. **鼓励为主**：每次成功组合给予正面反馈（撒花动画 + 文字鼓励）
4. **离线友好**：图标和字体本地部署，不依赖 CDN
